Introducción a las Bases de Datos

Publicado por Marcelo Freri el 3/3/2023

Las bases de datos están presentes en nuestra vida cotidiana más de los que nos damos cuenta. Tu lista de contactos en el móvil, las series que sigues en tu servicio de streaming favorito, tus publicaciones en redes sociales, tus correos electrónicos y hasta tu historial de vacunación se encuentran almacenados en bases de datos.

A continuación realizaremos un introducción al mundo de las bases de datos, para que puedas tener una idea general de para qué y cómo se utilizan en el ámbito informático.

¿Qué es una Base de Datos?

Una base de datos es una estructura que permite el almacenamiento de información, de manera ordenada y metódica, y que agiliza luego su consulta.

Componentes de una Base de Datos

Algunos de los componentes principales de una base de datos son:

  • Tablas
  • Índices
  • Reportes

Tablas

Las tablas son el corazón de la base de datos, pues en ellas se almacena el grueso de la información. Piensa en ellas como en hojas de una planilla electrónica con largas listas de datos tabulados.

Un ejemplo práctico de una tabla, sería la lista de contactos de tu teléfono móvil.

Internamente, una tabla se organiza en campos y registros.

Campos

Es lo que la da orden a la información volcada en una tabla, etiquetando y dando formato a cada dato almacenado.

Volviendo al ejemplo de la agenda de contactos del móvil, los campos serían nombre, apellido, teléfono, etcétera.

Un campo determina qué dato se va a almacenar en él y también su tipo. Si es texto, número o fecha lo determina el tipo de campo. Estos que nombré no son los únicos tipos que hay, existen muchos más.

Registros

Si los campos determinan la información que se va a contener dentro de una tabla, los registros son la información en sí.

En el ejemplo de la agenda de contactos cada contacto, con sus datos asociados, correspondería a un registro de la tabla. En una tabla pueden existir millones de registros.

Ejemplo de una tabla.

Índices

Un índice es método usado para agilizar el acceso a la información de una tabla. Básicamente son como tablas auxiliares que contienen una copia de los campos más utilizados de una tabla.

El administrador de la base de datos (DBA por sus siglas en inglés) debe determinar cuáles son los campos más accedidos y crear índices sobre esos campos. Al tratar con un menor volumen de datos, las consultas se realizan en mucho menor tiempo si se accede a través de índices.

Para un usuario los índices son una herramienta completamente transparente. Desde su punto de vista, sólo debe acceder a la información, sin importar si lo hace mediante un índice o directamente sobre la tabla, ya que es tarea del DBA definirlos según estime necesario.

Reportes

Como su nombre lo indica, un reporte es un informe que se arma utilizando la información contenida en la base de datos. Por lo general, un reporte suele utilizar datos de varias tablas para armar un informe.

¿Qué es una Base de Datos Relacional?

La tabla de contactos no es la única tabla que compone la agenda de tu móvil. Por ejemplo, hay una tabla que registra las llamadas ocurridas. Esta tabla probablemente contenga campos con la fecha de la llamada, el número de teléfono, si se produjo la conexión o si fue una perdida. Esta tabla está relacionada con la tabla de contactos, posiblemente por el número de teléfono. Es por esto que puedes ver el nombre del contacto que te llama.

Esta forma de conectar un registro de una tabla con uno o varios registros de otra es lo que denominamos una relación. Entonces, una base de datos cuyas tablas tienen relaciones definidas entre ellas es una base de datos relacional.

En la práctica, casi todas las bases de datos que se usan en la actualidad en informática son relacionales.

En el ejemplo de la imagen, las columnas corresponden a los campos y las filas de datos a los registros. La flecha indica que existe una relación entre los campos teléfono de cada tabla.

Claves Primarias y Secundarias

Un componente omnipresente en las bases de datos relacionales son las claves. Se trata de campos que se utilizan para relacionar tablas entre sí.

En el ejemplo de la agenda telefónica, el campo teléfono sería la clave, pues sirve para relacionar entre ellas la tabla de contactos y la de llamadas.

Una clave primaria es la que identifica un único registro en toda la tabla, es decir, no pueden haber más registros que tengan la misma clave. En el caso de los contactos, el campo teléfono sería una clave primaria.

Que el campo teléfono sea una clave primaria es a modo de ejemplo, ya que en la realidad es posible tener dos o más contactos con el mismo número telefónico. Por ahora, sígueme la corriente.

En el caso de la tabla de llamadas, el campo teléfono puede aparecer varias veces, sin embargo sigue sirviendo para relacionar esta tabla con la de contactos. En esta oportunidad hablamos de una clave secundaria; una clave que se usa para relacionar tablas pero que no es única.

Como se desprende de todo esto, las relaciones pueden ser de una a una o de una a muchas; esto dependerá de si un registro tiene un único registro equivalente en otra tabla o si, por el contrario, tiene más de uno.

Sistemas Gestores de Base de Datos

Cuando una aplicación accede a una base de datos, no lo hace directamente sobre los archivos que la componen, sino que envía o solicita la información a través de un programa que se encuentra en el servidor de bases de datos. Este software se llama Sistema Gestor de Bases de Datos y permite realizar todo tipo de tareas sobre una o varias bases de datos y sus componentes (creación, acceso, modificación, etc). Además, debe encargarse de la integridad y consistencia de la información tratada.

Cuando accedes a unos datos que están indexados, es el gestor de bases de datos quien se encarga de utilizar el índice adecuado. También se encarga de mantenerlos actualizados cuando se agrega, modifica o elimina algún registro.

Otra tarea importante que realizan los gestores es encargarse de controlar los accesos a los datos a través de políticas de usuario. Los DBA deben definir la seguridad de la base de datos en función de las tareas que realiza cada usuario, luego el sistema se encargará de gestionar los permisos.

Como último punto, pero no menos importante, los sistemas gestores de bases de datos suelen ser utilizados para realizar los respaldos de las bases de datos que manejan.

Algunos de los gestores de bases de datos más utilizados son los siguientes:

  • Oracle
  • MySQL
  • MS SQL
  • PostgreSQL

SQL

Como habrás observado, muchos gestores de bases de datos llevan SQL en su nombre. Esto tiene un motivo, además de la falta de imaginación, y es el hecho de que soportan el Lenguaje Estructurado de Consultas (Structured Query Language), abreviado SQL.

Se trata de un lenguaje diseñado para la interacción con bases de datos. No es el propósito de este artículo enseñarte SQL, pero vamos a mirar un ejemplo rápido.

 1# Creo la base de datos
 2CREATE DATABASE pruebas;
 3
 4# Abro la base de datos
 5USE pruebas;
 6
 7# Creo una tabla llamada "usuarios" con campos "id", "usuario", "nombre" y "activo"
 8CREATE TABLE usuarios (id INT NOT NULL AUTO_INCREMENT, usuario VARCHAR(20) NOT NULL, nombre VARCHAR(45) NOT NULL, activo BOOL NOT NULL DEFAULT True, PRIMARY KEY (id));
 9
10# Agrego datos a la tabla "usuarios"
11INSERT INTO usuarios (usuario, nombre) VALUES ("jfernandez", "Juana Fernández");
12INSERT INTO usuarios (usuario, nombre) VALUES ("jrodo", "José Rodó");
13INSERT INTO usuarios (usuario, nombre) VALUES ("pfigari", "Pedro Figari");
14INSERT INTO usuarios (usuario, nombre) VALUES ("dagustini", "Delmira Agustini");
15
16# Modifico un campo de un registro de la tabla
17UPDATE usuarios SET activo=False WHERE usuario="pfigari";
18
19# Muestro todos los datos de la tabla "usuarios"
20SELECT * FROM usuarios;

En SQL los comandos se escriben en mayúsculas y los parámetros en minúsculas. Esto es una antigua convención, aunque la mayoría de los gestores modernos ya aceptan minúsculas para los comandos.

Conclusión

Las bases de datos están presentes en la mayoría de los ámbitos que requieran el procesamiento de grandes volúmenes de información. Están compuesta por varios elementos, destacándose principalmente las tablas e índices.

Un sistema gestor de base de datos es un software que se encarga de administrar las bases de datos, asegurando integridad en el manejo de la información, manteniendo los índices actualizados y controlando los permisos de los usuarios, entre otras tareas.

SQL es un lenguaje que permite comunicarse con el gestor de bases de datos para realizar consultas y modificaciones.
